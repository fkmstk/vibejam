# AGENTS.md

## プロジェクト概要

### 目的

Vibe Coding Game Jam 向けに応募可能な新作 Web ゲームを制作し、賞金獲得を狙う。

### コンテスト概要

- 名称: 2026 Vibe Coding Game Jam
- 賞金:
  - Gold: $20,000
  - Silver: $10,000
  - Bronze: $5,000
- 応募締切:
  - 2026-05-01 13:37 UTC
  - 日本時間: 2026-05-01 22:37 JST

## 参加条件

- 誰でも参加可能
- 1人1作品まで
- ゲームは新作限定
- 2026-03-24 以前から存在していたゲームは不可
- コードの 90%以上を AI が書いていること
- Web で遊べること
- 無料公開であること
- ログインやサインアップ不要で遊べること
- 重いダウンロードや長いロード時間は避けること
- マルチプレイは推奨だが必須ではない
- エンジンは自由
- Three.js 推奨
- entrant badge 用スクリプトの導入案内あり

## 実装要件

- ブラウザで即プレイ可能
- 初回ロードをできるだけ軽量化
- モバイルでも最低限プレイ可能
- URL共有だけで遊べる構成
- サーバ不要または極小構成を優先

## 開発方針

- AI 主導開発を前提にする
- 人間側の主担当:
  - 企画
  - ゲーム性の絞り込み
  - 最終品質確認
  - バランス調整
  - デプロイ
- 開発ログを残し、AI が主要コードを書いたことを説明できるようにする

## おすすめ技術スタック

### 最有力

- フロントエンド: TypeScript
- 描画: Three.js または Phaser
- ビルド: Vite
- 配信: Vercel / Cloudflare Pages / Netlify

### 軽量寄り代替案

- Canvas 2D + TypeScript + Vite

## 企画条件

企画は以下を満たすのが望ましい。

- 3秒以内に面白さが伝わる
- 1分以内でルール理解できる
- 配信やSNSクリップ映えする
- AI生成の強みが活きる
- 1週間前後で完成可能なスコープ

## 企画候補

### 1. 物理演算カオス系

- 例: 吹っ飛ばし、崩壊、連鎖爆発
- 強み: 一目で分かる、配信映え

### 2. 群衆制御サバイバル系

- 例: 逃げる、集める、進路を作る
- 強み: Webで軽く作りやすい

### 3. 1ボタン高難度アクション

- 例: タイミング回避、落下回避
- 強み: 開発が短期で済む

### 4. AIっぽさを演出した生成変化ゲーム

- 例: 毎回マップや敵配置が変わる
- 強み: ジャムの文脈に合いやすい

## 評価されやすそうな方向

- ルール説明なしでも理解できる
- 画面を見た瞬間に違いが伝わる
- リトライしたくなる短いプレイサイクル
- バグが少ない
- ロードが速い
- 触ってすぐ楽しい

## やらない方がよいこと

- 大規模RPG
- 説明文を読まないと分からないゲーム
- ダウンロード必須構成
- アカウント作成必須
- アセット過多で重い構成
- ネット同期が複雑な本格マルチプレイ

## 開発タスク

### フェーズ1 企画

- コア体験を1文で定義
- 参考ゲームを3本集める
- 1プレイ時間を決める
- 勝敗条件を決める

### フェーズ2 試作

- 24時間でプロトタイプ作成
- 面白さ判定
- 重さ計測
- スマホ動作確認

### フェーズ3 仕上げ

- UI最小化
- 音追加
- チュートリアル追加
- バグ修正
- ロード時間最適化

### フェーズ4 応募準備

- 公開URL準備
- entrant badge導入
- 応募文作成
- AI利用説明の整理

## 直近の推奨アクション

1. 企画を3案に絞る
2. 一番軽く作れて一目で面白い案を選ぶ
3. Vite + TypeScript で雛形作成
4. まずPCブラウザ版を完成
5. その後スマホ最適化
6. Vercelなどへデプロイ

## このプロジェクトの勝ち筋

- 短期間で完成する
- Webで軽快に動く
- AI主導開発の条件に素直に合う
- 見た瞬間に面白さが伝わる
